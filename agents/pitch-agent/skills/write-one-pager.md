# Skill: Write One-Pager

## When to invoke
Task involves summarizing the built MVP for customers, partners, or investors.

## Steps
1. Open with the problem in one sentence a non-technical reader nods at
2. Describe the solution in terms of what the user does, not what the code does
3. List the three strongest built features — only features that actually exist in the output
4. Name the target customer specifically ("independent salon owners", not "SMBs")
5. Close with one call to action

## Output format
// filepath: report/pitch-one-pager.md
One page maximum. Plain language, no engineering jargon, no invented features.
