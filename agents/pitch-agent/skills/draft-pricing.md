# Skill: Draft Pricing

## When to invoke
Task involves proposing what the small business should charge for the MVP.

## Steps
1. Anchor pricing to the customer's alternative (doing it manually, hiring someone, a bigger competitor tool)
2. Propose two or three tiers maximum; name each tier by who it's for
3. Gate tiers on value metrics visible in the built features (users, volume, integrations) — never on features that don't exist
4. If run-cost estimates are available in the digest, sanity-check that the lowest tier covers per-user run-cost
5. Mark all numbers as starting points to validate with the first five customers

## Output format
// filepath: report/pricing-draft.md
Tier table plus a short "why these numbers" paragraph. Starting points, not final pricing.
